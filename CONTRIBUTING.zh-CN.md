# 贡献指南（中文版）

## 欢迎！

互联网广域世界（Web Wide World）是一个去中心化共享虚拟世界协议。我们非常欢迎任何形式的贡献，包括代码、文档、设计、讨论、推广……等等。

## 如何参与

### 1. 报告问题或提出建议
- 使用 [Issues](https://github.com/nobody-and-everybody/web-wide-world/issues) 页面
- 中文问题请添加 [`lang/zh-CN`](https://github.com/nobody-and-everybody/web-wide-world/labels/lang/zh-CN) 标签
- 如果不知道用什么标签，直接写中文即可，我们会处理

### 2. 提交代码更改
- **工作流程**：Fork → 创建功能分支 → 测试 → 提交 PR
- **代码规范**：请遵循现有代码的风格和结构
- **测试**：尽量为你的修改添加测试用例
- **提交信息**：建议中英文皆可，但需清晰说明改动内容

### 3. 完善文档
- 发现错别字、表达不清、缺少示例，可直接提 PR
- 文档现有语言：
  - 英文：主 README.md 和 `docs/en/`
  - 中文：README.zh-CN.md 和 `docs/zh-CN/`
- 欢迎协助其他语言的翻译

### 4. 推广与社区建设
- 在博客、技术平台、社交媒体分享你对项目的理解
- 帮忙完善中文解释材料（如：在知乎、掘金、B站发布教程）
- 组织或参与相关在线讨论

## 分支策略

- `main`：稳定版本，通过 PR 合并
- `feature/*`：新功能开发分支
- `docs/*`：仅涉及文档的修改
- `lang/zh-CN/*`：中文相关翻译或内容更新

## 提交 PR 步骤

1. Fork 本项目
2. 创建分支：`git checkout -b feature/your-feature-name`
3. 提交修改（`git commit -m '简短描述'`）
4. 推送到你的 fork：`git push origin feature/your-feature-name`
5. 在 GitHub 上点击 “New Pull Request”
6. 填写标题与详细说明，关联相关 Issue（如有）

## 中文社区特别通道

- 如果你不熟悉 GitHub 协作流程，可以先在本仓库的 [Discussions](https://github.com/nobody-and-everybody/web-wide-world/discussions) 中发起话题（可使用中文）
- 也可以直接通过 [Issues](https://github.com/nobody-and-everybody/web-wide-world/issues) 发问（可加 `help` 和 `lang/zh-CN` 标签）

## 行为守则

我们遵守 [贡献者公约](https://www.contributor-covenant.org/zh-cn/version/2/1/code_of_conduct/)。请保持友好、尊重、包容的态度，让每个人都能在项目中感到安全与被支持。

## 需要帮助？

如果你在贡献过程中遇到任何困难，随时可以通过 Issue 或 Discussion 联系项目维护者。我们希望能帮助你顺利参与进来！

## 致谢

感谢每一位贡献者的时间与精力。无论你的贡献是大是小，都是构建这个共享世界的重要一砖一瓦。
